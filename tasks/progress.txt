# Ralph Progress - Freespace TUI MVP

## Codebase Patterns
- Rust project using edition 2021 with ratatui + crossterm for TUI
- Dependencies: ratatui 0.29, crossterm 0.28, tokio (full), clap (derive), toml, serde (derive), walkdir, glob, humansize, anyhow, thiserror
- Scanner pattern: `scanner::start_scan(modules, tx)` spawns a `tokio::task::spawn_blocking` thread. Event loop calls `process_scan_messages()` with `try_recv()` each tick to drain updates non-blockingly.
- Channel: `tokio::sync::mpsc::unbounded_channel()` carries `ScanMessage` enum (ItemDiscovered, ModuleComplete, ModuleError, ScanComplete)
- Path expansion: `expand_tilde()` replaces `~` with `$HOME`, then `glob::glob()` expands wildcards
- Size calculation: `walkdir::WalkDir` to sum file sizes in directories; `std::fs::metadata().len()` for files
- main() uses `#[tokio::main]` to provide async runtime for `spawn_blocking`
- Use `cargo check` for typecheck verification
- Stub files must use `//` comments, NOT `///` doc comments (doc comments without an attached item cause compile errors)
- `Tui` type alias in src/tui/mod.rs: `Terminal<CrosstermBackend<Stdout>>` — use this throughout the app
- Terminal lifecycle: call `tui::install_panic_hook()` then `tui::init()` on startup, `tui::restore()` on exit
- main() returns `anyhow::Result<()>` for `?` operator usage
- Theme is on `App.theme` — use `app.theme.style_*()` methods in view render functions for consistent styling
- Use `Color::Indexed(n)` (256-color palette) for terminal-compatible colors
- Size formatting: `tui::widgets::format_size(u64)` and `format_size_or_placeholder(Option<u64>)` — use in views for displaying sizes
- Module TOML format: top-level fields (name, version, description, author, platforms) + `[[targets]]` array with path/description. Paths use `~` and `*` globs.
- Built-in modules live in `modules/<name>/module.toml` — parsed via `Module` struct in `src/module/manifest.rs`
- Event loop in `App::run()`: render → poll (250ms tick) → handle_key → loop. `should_quit` flag breaks the loop.
- View rendering dispatched via `match self.current_view` in `App::render()`. Each view has its own render method on App.
- Key handling dispatched via `match self.current_view` in `App::handle_key()`. Global `q` quits before dispatching to view handlers.
- Layout pattern: 3 vertical chunks (title bar Length(3), content Min(1), status bar Length(1))
- Module loading: `manager::find_modules_dir()` returns `Option<PathBuf>`, `manager::load_builtin_modules(&Path)` returns `(Vec<Module>, Vec<String>)` (modules + warnings)
- Platform filtering: `env::consts::OS` returns "macos"/"linux"/"windows" — matches module manifest `platforms` field directly
- View rendering pattern: view files in `src/tui/views/` export a `render(app: &App, frame: &mut Frame)` function; `App::render_*` methods delegate to them
- Use `Table::row_highlight_style()` (not deprecated `highlight_style()`) for row selection in ratatui 0.29
- `sorted_module_indices(&App) -> Vec<usize>` in `module_list.rs` provides stable sort by total_size desc — reuse in key handlers
- `selected_index` on App is a visual row position in the sorted list, not a module index — map via `sorted[selected_index]`
- `sorted_item_indices(app, module_idx)` in `module_detail.rs` provides stable sort by item size desc — reuse in key handlers for detail view
- Detail view key handler: extract `module_idx` from `View::ModuleDetail(idx)` via pattern match; use `sorted_item_indices()` to map visual position to item index
- Space toggle pattern: `HashSet::remove()` returns bool — use it to check if item was already selected; if not, insert it
- Modal/overlay pattern: use `Clear` widget + `centered_rect()` helper to render a dialog on top of the current view
- `collect_selected_items(app)` in `cleanup_confirm.rs` gathers all selected items across modules — reuse for cleanup operations
- `previous_view: View` on App tracks return destination from overlay views — set before transitioning to CleanupConfirm or Help
- `cleaner::delete_items(&[PathBuf])` returns `CleanupResult { succeeded, failed }` — use for live cleanup operations
- View enum derives `Clone, Copy` — can be stored/compared freely
- Spinner animation: `app.tick_count % SPINNER_CHARS.len()` indexes into a braille char array; tick_count incremented each event loop iteration via `wrapping_add(1)`
- Minimum width check (80 cols) is centralized in `App::render()` — individual views do NOT need their own width checks
- `Event::Resize` uses `continue` in the event loop for immediate re-render
